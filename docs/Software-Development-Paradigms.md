### *B.　Compare Matrix*
| 縮寫 | 完整名稱 | 核心定位 | 關注焦點 | 主要面向對象 |
| :--: | :-- | :--: | :-- | :-- |
| **BDD** | *Behavior-Driven Development* | **行為驅動開發** | 系統的商業行為與用戶體驗是否正確 | PM / QA / RD 跨職能團隊 |
| **DTT** | *Decision Table Testing* | **判定表測試技術** | 複雜多條件組合下是否有邏輯遺漏 | QA / 開發者 (規劃邊界測試) |
| **TDD** | *Test-Driven Development* | **測試驅動開發** | 代碼的內部設計與架構是否乾淨 | 開發者自己 (防改 Code 壞掉) |
| **TBD** | *Trunk-Based Development* | **主幹開發模式** | 代碼分支策略與頻繁整合 (CI/CD) | 全體開發團隊與運維人員 |
